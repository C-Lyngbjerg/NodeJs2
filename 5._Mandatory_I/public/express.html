Express

It is the way we create an API server in Node.js. It is similar to Spring from Java, but requires a lot less set up and unlike Spring doesnt require things like Thymeleaf (as far as i know from the time of writing). It has various methods to serve HTTP requests and serving html files in response to the requests.

When you want to set up a node application with Express. The first thing we do is create package.json where we write the key:value pair


{"dependencies": {"express":<version>}}


After which we run npm install which installs all our specified dependencies - in this case only Express, but we would probably also use Cross-Env. After npm has finished we can see a new file called package-lock.json and a folder named node_modules which together holds all we need to create our Data API server. The last step i will describe in this segment is importing express and using it in app.js.


const express = require("express");
const app = express();
app.use(express.json());


These three lines we put in the top of our new app.js file. The first is a statement to import express, the second is to start using it and the third is so we can serve .json files as responses to the HTTP requests.

Methods such as app.listen() and app.get() will be explained in the following segments.

GET, app.listen etc

HTTP Requests

HTTP stands for Hyper Text Transfer Protocol. It is what we use send data from our API. There is a order to to which we set up our Express methods for these HTTP requests: GET, which is where our server responds with all resources or if by a specific criteria eg. id, our server responds with a resource that corresponds with that criteria.
The next is POST, which is where the client creates a new resource in our server - here we usually use Postman or a form on a HTML page.
PUT/PATCH is used for updating all or one specific resource respectively - It is a good idea to look at our array methods to easily find and update the specific resource. Here again the usage is the same as with POST.
The last is DELETE, which is used for deleting a specific resource. We will almost always use DELETE with a specified criteria eg id, because we dont want to delete our whole database.


Express Methods

These following methods are how we connect the HTTP requests with our application and how we make sure our application runs.

app.get()

As mentioned above these app.<request> methods are how we connect our server to HTTP requests. Below is my example method which is this projects get method for the index page. The first line is the function declaration with two parameters. The first is the URL enclosed by quotation marks and the second being an arrow function which in turn takes req and res objects as parameters. Inside the arrow function we create a variable which contains the name our .html page for our index page and then the res object is used to send the path to the file as a response combined with our variable which contains the path to the public folder


app.get("/", (req, res) => {
const filename = "/index.html"
res.sendFile(pubDir +filename);
});


app.listen

This method is always - by convention rather than function - written at the bottom of the app.js file. The method is used to make sure that our application is continuesly listening on the specified


app.listen(port, (error) => {
if (error) {
console.log("Couldn't run server: ${​​error}");
} else {
console.log("Server is running on port", port);
}});


This method - like the request methods above contains two parameters, the first being an optional error and the second an arrow function with an if/else statement that either prints the error or prints which port the server is now running on.

An endpoint is where an html page or some API service is exposed to users. For this project the endpoints would be "/", "/express" etc.

Redirecting is when we, instead of just sending the user to the html page or sending back another requested resource our server responds to the query by immediately, or at a set time, Redirecting the user to another resource by using the res.redirect instead of .send as usual to move the user from the current resource to the specified one.


res.redirect("/");
