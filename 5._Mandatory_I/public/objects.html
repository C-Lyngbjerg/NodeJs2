Objects & Variables

In Node.JS we use let og const when we want to declare a variable. When we want to use a mutable variable we use let and when we want to use an 'immutable' variable we use const. there is also var, but i will touch on why we dont use var in the next section.

Immutable was in quotes, because it is actually possible to fx change the value of a Const object even though we call it 'immutable'. An example can be seen below:


const thisSong = {"description": "The best song in the world."};
delete thisSong.description;


This code would result in an empty thisSong object, even though it is 'immutable'. However something like the following code would result in a compilation exception.


const x = 1;
x = 2;


So as a simple variable const are actually immutable, but as objects the same doesnt quite apply

Var is as mentioned above in the let & const-section not used anymore in JS development for the following reason: scope. Scope is, simply put, the 'area' of code in which a variable or function is accesible.

So the difference between var compared to let and const is that var has 'function scope' where let and const have 'block scope', which means that they are only accesible within a block of code for example between curly braces {...} in an if-statement.

A var on the other hand has 'function scope', which means that it is accesible from the entirety of the scope of the function which it is inside. This means that you have to be a lot more careful with your naming of variables and what you do with them, because you risk overwrting an already declared variable or something similar.

An Array is the basic datastructure we used the most in Node.JS. To initialize an Array we use the following statement


const newArr = [1, 2, 3]


Arrays are 0 based indexing as we are used to and to add to an array we use .push to add to the end of the array most often or add by index. There are many more array methods that can be easily googled and many are familiar to someone who is used to Java.


Methods

There are many more array methods than described below, but i will explain three of the most important and different from Java. The rest can be easily googled and should be familiar to someone who is used to Java.

.splice()

A very useful method which is .splice that can be used to insert, delete or replace by index. The method returns an array of deletions, which means that if no deletions are made .splice returns an empty array


newArr.splice(<start>, <count of deletions>, <item 1>, ... , <item n>);


.filter()

.filteris preferable to use over a for-each loop, because the latter just iterates over each element in an array while the former returns a new array and iterates through elements with a provided callback method. It uses the callback function to check if each currentValue equates to true compared to the requirements of the function. If true it is added to the array that is eventually returned.


const newerArr = newArr.filter(<currentValue> => <requirements>);


.map() & spread

The last method i will explain is the .map which is similar to the previous, but instead of returning an array of the elements which pass the test, this returns an array with the result of the callback function. This means that a function like


const newerArr = newArr.map(<currentValue> => <currentValue> * 2);


Will return an array containing each element multiplied by two. This is very useful when you want to for example want change one or more element in an array. For example with the spread operator (...<newArr>). The spread operator works on either arrays or objects and it returns the array/object as each of it's elements or key:value pairs. If you use more than one spread operator and the objects have the same keys, the following spread will overwrite the previous making it very useful in a situation such a Patch request, where you want to ensure that the client fx doesn't enter a destructive id value


return {...animeName, ...req.body, id: animeName.id}


Here the first spread which is animeName will have it's values overwritten by the body of the patch request, but if there are missing key:value pairs then they just wont be replaced in animeName. This is better than just replacing the entirety with req.body, because then you would always have to enter every other key:value as well as the one you were interested in changing.
The final thing that happens is id: animeName.id where we ensure that no matter what the id wont ever be changed clientside.

JSON stands for JavaScript Object Notation and is the way we in Node.JS write our objects. It is also used in Express in the package.json where we write our dependencies and scripts and is also the datastructure used by our APIs.

A JSON is always begun by {} and is made up of key:value pairs. For example in the package.json we write our dependencies like this:


{
"dependencies": {
"express": "^4.17.1"}
}


Because a JSON object is made up of key:value pairs, we can call and manipulate them with calls like


const JSONExample = {key: "value"};
JSONExample.key = otherValue;
JSONExample.newKey = newValue;


An enviroment variable can be many different things in an application, but the most important we've used is process.env.PORT, which is the port that our application is run on. We use this variable when we run the command npm run start-dev, which is an npm command which also uses the cross-env package.


"start-dev": "cross-env PORT=3000 nodemon app.js"
