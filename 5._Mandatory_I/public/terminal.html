Terminal Commands

git is what we use as our Version Source Control. It is a language which is made to ensure that when several developers are working on the same project there arent any problems.
there are a few main commands that we use from GIT:


  • git add -A <&/ File>

  • git commit -m <Commit Message>

  • git push

  • git pull <Named Repository &/ Branch>

  • git init

  • git status

  • git clone <URL>


Most of these are pretty selfexplanatory, so i wont go in to much detail, but always remember to follow the correct order eg.


  1. git add -A

  2. git commit -m <Commit Message>

  3. git push


Otherwise it's really easy to mess something up.

A more comprehensive list of common git commands can be found here

npm stands for Node Package Manager. It is the way we install different packages in Node.JS such as Express, nodemon and Cross-Env. It is also the way we run a lot of scripts and terminal commands. Whenever you clone a Node.JS application from github or the like, the first step is to run the below command to make sure all dependencies are installed.


npm install


Can also be written with i instead if you feel it's too much work to write out install.
The other command we use a lot is


npm run start-dev


This command is made up of a run call from npm and the last part is the name/key of the script in package.json that we want to run.

Nodemon is a tool that automatically detects changes in the code and restarts the application whenever there is a pause in the changes. It is installed with npm which is explained more in the above section. As with any terminal command it is important to cd into the correct directory before you start trying to run an app that the terminal cant access.

There are a few other useful terminal commands that dont fit into the other categories. They are all fairly easy, but very useful for traversing your machine through the terminal.
The first is ls which is used for listing the files and folders in the current working directory. Apropos the next is pwd which prints the current working directory.

Next we come to cd < .. / folder> which changes directory to one of the following:

  • blank changes to home directory
  • .. changes to one step further up
  • <folder> changes to that folder - you can also specify a path with cd folder/subfolder

The final two commands i will touch on is how to create a file (just remember to write the filetype as well as the name touch <file.type>) and mkdir <foldername> which will create a folder with the specified name in your current working directory.
