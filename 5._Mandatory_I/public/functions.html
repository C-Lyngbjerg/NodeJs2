Functions

What is it?

Hoisting is when Javascript, at the time of execution, moves all variable and function declarations to the top of their respective scopes. However it is only the declaration and not the initialization of them that are moved, so if we try to print out a variable before we write it in the code it'll just return undefined, because it isn't initialized yet.

Functions

Hoisting also happens with functions, but only if they've been declared as functions - like below:


function myFirstFunction (){
console.log("Hello class");
}


In this case we would be able to call the function before we actually declare it in the code. If we declare it as a const or let the same would not apply - the application would throw an exception.

What to do about it

We can use what is called strict mode which is written at the top of the code in quotation-marks. This is a mode which is native to JavaScript which prevents things like hoisting, because it is easy to make a mistake and try to use an uninitialized variable or something similar.
